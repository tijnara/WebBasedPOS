# Personal Preferences

- **Code Style**: Prefer functional programming patterns and arrow functions.
- **Naming**: Use descriptive variable names; avoid abbreviations like `idx` or `err`.
- **Formatting**: Always use Prettier-style formatting (semi-colons, single quotes).
- **Communication**: Keep explanations technical and concise. Only provide step-by-step guides for complex setup tasks.
- **Tech Stack**:
  - Frontend: Next.js (App Router), Tailwind CSS.
  - Backend: Supabase, Server Actions.
  - State: React Context or Jotai for global state.
